Why this matters: By transforming survival data into a long-format table, any standard ML algorithm (random forest, XGBoost, neural network) can be applied directly — no custom survival-specific loss function needed. The reduction is valid because the Poisson likelihood on the expanded data equals the original survival likelihood.

Competing risks — two causes, one dataset expansion

This animation walks through Table 4 from Reduction Techniques for Survival Analysis (§6.2), using three patients from the sir.adm ICU dataset. Each patient can experience one of two competing events: discharge alive (k=1) or death (k=2).

i=30238 — discharged (k=1) at day 26, pneumonia, age 38.7, M
i=41 — discharged (k=1) at day 4, no pneumonia, age 75.3, F
i=17058 — died (k=2) at day 22, pneumonia, age 62.2, M
Discharge alive (k=1) — blue box when d_ijk=1
Death (k=2) — red box when d_ijk=1
Censored for this cause — green box, d_ijk=0
Key idea for competing risks: The dataset is duplicated — one copy per cause. In the k=1 (discharge) dataset, deaths are treated as censoring (d_ijk=0). In the k=2 (death) dataset, discharges are treated as censoring. Both datasets are then stacked with a cause column added, so a single binary classifier learns shared and cause-specific effects through interactions with the cause variable.
